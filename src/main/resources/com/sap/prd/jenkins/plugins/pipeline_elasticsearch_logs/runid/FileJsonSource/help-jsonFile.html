The JSON Run ID provider generates the Run ID for each run based on the JSON template defined in the file with the absolute path defined here.

You can use $variable or ${variable} for string values. Those values will be replaced by the corresponding environment variable if it exists. It is also possible to use pipeline/job parameters.

Most common variables are:

  • JOB_NAME (Jenkins native)
  • BUILD_NUMBER (Jenkins native)
  • JENKINS_INSTANCE_ID (injected by the plugin)
  • RUN_UID (injected by the plugin)

An example JSON string could look like this: 

{
    "jenkins-instance": "${JENKINS_INSTANCE_ID}",
    "build": {
        "project": "${JOB_NAME}",
        "build-number": "${BUILD_NUMBER}",
    },
    "branch": "$JOB_PARAMETER_BRANCH_NAME",
    "uid: "${RUN_UID}"
}
